# User Scenarios

## 1. 添加资料

用户把一段文本、一个完整本地路径或一段 AI 对话放进 GUI，点击 `添加资料`。系统调用 `organize`，写入 Obsidian 新整理记录；如果是路径，则生成索引清单和 manifest，只记录来源和建议，不复制、不移动源文件。

## 2. 搜索回顾

用户输入关键词或问题，点击 `搜索回顾`。系统调用 `review`，检索 Obsidian、助手报告、AI 归档和旧资料索引，返回本地摘要、匹配来源、相关原因和路径；无匹配时返回未完成，不编造答案。

## 3. 生成 AI 上下文包

用户准备继续问 AI，输入当前任务并点击 `预览候选来源`。系统调用 `extract` 的 preview 模式，只返回候选来源；用户点击 `确认生成上下文包` 后，系统再写入 AI 上下文包，包含来源路径、压缩摘要、安全边界和下一步请求。

## 5. 工具维护页

低频能力进入 `/advanced`：`file-radar` 查看文件雷达，`obsidian-health` 检查知识库，`legacy-index` 二次整理旧资料索引，`open-guidebook` / `open-interaction-guide` 打开教程和交互说明。它们只生成报告、索引或打开本地资料，不移动旧文件。
