# Guidebook

本目录保存可直接给用户阅读或导入 NotebookLM 的完整教程资料包。教程已按新主页同步：GUI 是可视化上下文入口，不是替代 Codex；核心产物是 Codex 接手包。

## 文件

- [knowledge-action-assistant-tutorial.pdf](knowledge-action-assistant-tutorial.pdf): 7 页 PDF 教程，覆盖从上下文入口到 Codex 接手包的主流程。
- [slides/page-01.png](slides/page-01.png): 产品定位：不是替代 Codex，而是减少 Codex 读上下文。
- [slides/page-02.png](slides/page-02.png): 本地文件、Obsidian、历史报告、AI 对话归档四类来源。
- [slides/page-03.png](slides/page-03.png): 如何生成 Codex 接手包。
- [slides/page-04.png](slides/page-04.png): 接手包里应该包含什么。
- [slides/page-05.png](slides/page-05.png): 记录与沉淀：Action、Card、Time、AI 对话归档。
- [slides/page-06.png](slides/page-06.png): 安全边界：默认只读，不删除、不移动、不重命名。
- [slides/page-07.png](slides/page-07.png): 一周使用节奏：轻量入口、周复盘、月度结构优化。

## 使用方式

1. 新用户先读 PDF，再打开 GUI 的“生成 Codex 接手包”。
2. 需要系统学习时，把 PDF 和 7 张分页图导入 NotebookLM。
3. 做演示或培训时，直接用 `slides/` 下的分页图逐页讲解。
4. 真要继续处理任务时，把接手包 prompt 复制给 Codex，而不是指望 GUI 替代 Codex。

## 安全边界

Guidebook 只解释使用流程，不要求用户把源文件交给工具自动移动、删除或重命名。助手默认只生成报告、建议、manifest 和明确的新笔记。
