# 2026-05-11 主题关联与知识详情回归迭代自查

## 本轮目标

延续上一轮 Mintlify / documentation-as-product 方向，把知识展示从“能展开”推进到“展开后更有用、更可验证”。

## 改动内容

- 关联内容从松散关键词重叠改为主题规则匹配。当前主题包括 Codex 会话、AI 上下文、Obsidian、本地文件、知识卡片、整理工作流、学习资料、本地化翻译、生活资料。
- 普通笔记详情改为阅读友好结构：`阅读摘要`、`关键信息`、`下一步`，不再复用知识卡的 `适用场景` 模板。
- `build_review_index` 保留完整 Markdown 到 `raw_markdown`，让知识详情能基于真实笔记正文提取段落，而不是只读压缩摘要。
- Playwright E2E 增加知识卡展开检查：必须看到 `一句话结论`、`关联内容`、`可追问问题` 和 `来源`。

## 九问自查

- 做到了视觉清晰阅读友好了吗？进一步改善。普通笔记现在按阅读摘要、关键信息、下一步组织。
- 做到了总结归纳到位，没有废话不遗漏关键信息了吗？比上一轮更好。完整 Markdown 进入知识流处理，减少摘要丢段落。
- 发给别人能看懂吗？更容易。相关卡片现在说明“同属主题”，不是泛泛共享关键词。
- 传到 Codex 指引性足够吗？增强。E2E 会强制检查可追问问题和来源，避免详情退化。
- GUI 双向交互方便吗？本轮仍偏展示质量，三主操作闭环保持不变。
- 信息内容容易编辑吗？来源路径仍保留，用户可回 Obsidian 原文编辑。
- 展示的方式足够方便吗？更方便。普通笔记不再套错模板。
- 信息集关联有做吗？已升级为主题关联第一版，不引入向量库。
- 能让人发散性思考吗？有。可追问问题继续作为发散入口，E2E 已纳入回归。

## 下一轮改进

- 主题规则仍是启发式，后续可增加用户可编辑的 `topics.yml`，让主题可配置。
- 搜索回顾结果页还可以复用同一套“主题关联 + 可追问问题”展示，而不是只在知识流详情里使用。
- 生成 AI 上下文包前的候选预览可显示主题命中原因，帮助用户判断是否选错来源。
