# Self Evolution Roadmap

本页记录下一版“Obsidian + AI 知识行动助手”的自我进化方向。目标不是增加更多扫描指标，而是让用户更容易开始、持续使用、复用沉淀内容，并在关键任务上被引导做更深判断。

## 1. 交互怎么更方便

助手首页采用场景入口，而不是把“扫描”放在第一位：

- “今天先干什么”只输出 1-3 个轻量重点，不把全部 backlog 变成当天任务。
- “记录一个任务”直接生成 Action 笔记，默认包含目标、背景、过程、结果、资料、下一步和验收标准。
- “这段内容放哪”先判断生活、学习、工作，再建议 inbox、daily、project、routine、archive。
- “调用知识索引”允许用户用自然语言找以前沉淀的 Action、Card、Time、X-AI 笔记。

后续优化重点：

- 把 GUI 返回结果拆成“建议动作”“打开路径”“复制 prompt”三种可点击输出。
- 给每个场景提供 1 个示例输入，降低第一次使用成本。
- 把高级扫描放到诊断区，避免新用户被指标淹没。

## 2. 安装部署初始化更快捷

初始化链路统一收敛到 `scripts/init-assistant.ps1`：

- 复制 `config.example.json` 到 `config.local.json`。
- 生成 Obsidian 使用指南和场景演示报告。
- 生成自我进化报告，方便用户理解当前能力边界。
- 提示用 `start-assistant-gui.ps1` 启动本地 GUI。

设计原则：

- 默认不删除、不移动、不重命名、不重写源文件。
- 私有路径只进入 `config.local.json`，不进入 Git。
- 首次使用优先跑 demo 和 guidebook，不要求用户理解全部配置项。

## 3. 怎么引领使用者深度思考

助手用 ACT 结构把“记录”升级为“行动、复用、复盘、交接”：

- Action: 追问为什么现在做、验收标准、第一步、来源和边界。
- Card: 追问复用条件、不适用边界、一句话结论、来源是否可靠。
- Time: 追问真正推进了什么、卡点原因、下一步是否足够小、哪些归档候选应延后。
- X-AI: 追问 AI 必须读哪些真实文件、安全边界、输出路径、失败时保留哪些证据。

这套追问要服务行动，不做空泛反思。日复盘保持轻量，周复盘再处理结构和 backlog。

## 4. 归纳内容怎么调用更方便

沉淀内容如果不能调用，就只是归档。下一版把知识调用拆成四步：

- 先用自然语言查询知识索引。
- 优先复用 Card 的结论和适用场景。
- 再用 Action 承接当前任务，避免只收藏不执行。
- 需要继续交给 Codex/OpenClaw 时，生成 X-AI 交接 prompt，带路径、边界、目标和验收标准。

默认输出应该包含：

- 匹配到的笔记路径。
- 为什么匹配。
- 推荐下一步。
- 是否需要创建新 Action。

## 验证标准

- GUI 能看到快速初始化、深度思考引导、调用知识索引、打开教程 PDF。
- `assistant_evolution.py report` 能同时写入 runtime 报告和 Obsidian 笔记。
- `project_quality.py` 能检查 guidebook PDF 和 7 张分页图是否已发布。
- `scripts/verify-harness.ps1` 覆盖新增自我进化测试。
