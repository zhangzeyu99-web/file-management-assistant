# GUI 交互说明

这份说明回答两个问题：哪里可以点，点击后页面会变成什么样。

![交互总览](assets/gui/interaction-map.png)

![点击后状态](assets/gui/interaction-states.png)

## 通用交互

所有按钮遵循同一条链路：

```text
输入内容或直接点击 -> 本地执行 action -> 右侧执行结果更新 -> 查看全部展开 JSON 详情 -> 继续复制 Prompt / 打开 Obsidian / 查看报告
```

默认安全边界不变：只读建议，不删除、不移动、不重命名、不重写源文件。只有明确的记录类按钮会写入新的 Obsidian 笔记。

## 页面状态

| 状态 | 用户看到什么 | 说明 |
| --- | --- | --- |
| 输入前 | 今日操作台为空，右侧显示最近状态 | 可以直接点，也可以先粘贴内容 |
| 点击中 | 按钮短暂禁用，避免重复提交 | 后端调用 `/api/action` |
| 完成后 | 右侧执行结果只显示本次动作 | 成功或失败都会写清楚 |
| 查看全部 | 下方黑色输出框展开 JSON 详情 | 用于排查路径、来源、Prompt |
| 继续使用 | 可复制上下文 Prompt，或打开 Obsidian | 不会自动移动或删除源文件 |

## 可点击入口

| 区域 | 按钮 | 输入要求 | 实际动作 | 页面结果 | 文件影响 |
| --- | --- | --- | --- | --- | --- |
| 今日操作台 | 问怎么用 | 可空 | `ask` | 输出针对当前问题的用法建议 | 不写文件 |
| 今日操作台 | 判断放哪 | 建议粘贴一段内容 | `inbox-route` | 返回生活/学习/工作分流和放置建议 | 不写文件 |
| 今日操作台 | 复制上下文 | 建议写当前任务 | `build-ai-context` 后复制 `prompt` | 输出可复制给 AI 的上下文 Prompt | 不写文件 |
| 新手区 | 打开教程 PDF | 可空 | `open-guidebook` | 打开本地教程 PDF | 不改源文件 |
| 新手区 | 查看交互说明图 | 可空 | `open-interaction-guide` | 打开本页图文说明 | 不改源文件 |
| 开始 | 今天先干什么 | 可空 | `today` | 只给 1-3 个今日重点 | 不写文件 |
| 开始 | 快速初始化 | 可空 | `onboarding` | 返回初始化步骤和命令 | 不写文件 |
| 开始 | 打开教程 PDF | 可空 | `open-guidebook` | 打开教程 PDF | 不改源文件 |
| 开始 | 查看交互说明 | 可空 | `open-interaction-guide` | 打开交互说明页 | 不改源文件 |
| 整理 | 查看文件雷达 | 可空 | `file-radar` | 返回最近扫描报告、候选、大文件等 | 只生成或读取报告 |
| 整理 | 这段内容放哪 | 建议粘贴内容 | `inbox-route` | 返回分流建议 | 不写文件 |
| 整理 | 检查知识库 | 可空 | `obsidian-health` | 返回收件箱、断链、空壳笔记等体检结果 | 只生成或读取报告 |
| 记录 | 记录一个任务 | 建议写任务目标 | `action-note` | 新建 Action 任务笔记 | 写新笔记 |
| 记录 | 沉淀知识卡 | 建议写可复用结论 | `card-note` | 新建 Card 知识卡 | 写新笔记 |
| 记录 | 复盘今天 | 建议写完成事项 | `time-review` | 新建 Time 复盘笔记 | 写新笔记 |
| 记录 | 归档 AI 对话 | 建议粘贴对话摘要 | `archive-ai-chat` | 新建 AI 对话归档笔记 | 写新笔记 |
| AI 续用 | 提取 AI 上下文 | 建议写当前任务 | `build-ai-context` | 输出来源路径、相关原因、压缩上下文、下一步请求 | 不写文件 |
| AI 续用 | 复制上下文 prompt | 建议写当前任务 | `build-ai-context` 后复制 | 复制可直接发给新 AI 对话的 Prompt | 不写文件 |
| AI 续用 | 打开 Obsidian | 可空 | `open-obsidian` | 打开本地 Obsidian vault 路径 | 不改源文件 |
| 右侧 | 刷新状态 | 可空 | `/api/status` | 刷新文件数、候选数、笔记数、报告时间 | 不写文件 |
| 右侧 | 查看全部 | 可空 | 前端展开输出框 | 显示最近一次 action 的完整 JSON | 不写文件 |

## 怎么判断该点哪个

| 你的目的 | 直接点 |
| --- | --- |
| 今天不知道先做什么 | 今天先干什么 |
| 不知道一段内容该放生活、学习还是工作 | 这段内容放哪 |
| 想把一个真实任务记下来 | 记录一个任务 |
| 想把一条经验留给以后复用 | 沉淀知识卡 |
| 想把今天轻量收尾 | 复盘今天 |
| 想保存一段已经发生的 AI 对话 | 归档 AI 对话 |
| 想让新 AI 对话先理解你已有上下文 | 提取 AI 上下文或复制上下文 prompt |
| 想检查知识库有没有乱 | 检查知识库 |
| 想看本地文件有什么要管 | 查看文件雷达 |

## 失败时看哪里

点击后如果右侧显示失败，先点“查看全部”。展开的 JSON 里会有 `error`、`path`、`sources` 或 `prompt` 等字段。需要继续让 AI 排查时，把这段 JSON 复制给 AI，比只描述“按钮没反应”更有效。
