# GUI 交互说明

GUI 是本地知识整理助手的可视化入口，不是临时工具页。它只负责把输入、来源、产物和下一步展示清楚，实际动作由 `knowledge_assistant.py` 的四个 action 执行。

![交互总览](assets/gui/interaction-map.png)

![点击后状态](assets/gui/interaction-states.png)

## 页面结构

- 顶部：说明这是本地知识整理助手，并展示安全边界、统一入口、搜索/筛选、来源可追溯和最近来源。
- 四张主卡：`整理资料`、`回顾知识`、`提取上下文`、`今日提醒`。
- 主输入区：一块统一输入框 + 快速场景 + 一个 `本地文件 / 目录目标` 区。
- 结果卡：默认展示人能读懂的结果，并固定拆成 `做了什么`、`来源是什么`、`产物在哪`、`下一步`。
- 右侧摘要：显示本地状态、最近来源 / 最近动作和安全边界。
- `高级/诊断`：旧能力入口，例如 `file-radar`、`obsidian-health`、`legacy-index`。

## 产品化交互原则

- 固定入口：用户不需要先理解 action 名，只要把内容放进统一入口，再选择四个主动作。
- 快速场景：按钮只负责预填真实可执行的整理、回顾、提取、提醒任务，不做伪操作。
- 搜索/筛选：回顾和提取先通过关键词/任务缩小范围，再显示来源和产物，避免把全部候选当成今日任务。
- 来源可追溯：结果区优先显示匹配来源、最近来源和产物路径，便于继续交给 AI 使用。
- 安全默认：扫描、回顾和诊断默认只读；整理、提取和提醒只写新记录。

## 主流程

```text
放入资料或问题 -> 点击四个主入口之一 -> 查看结果卡 -> 打开产物或复制 AI 上下文包 -> 需要时进入高级/诊断
```

## 可点击入口

| 按钮 | 输入 | 实际 action | 结果 | 文件影响 |
| --- | --- | --- | --- | --- |
| 整理资料 | 文本 / 路径 / AI 对话 | `organize` | Obsidian 新整理记录、来源、领域判断、下一步 | 只写新笔记 |
| 回顾知识 | 关键词或问题 | `review` | 本地摘要、匹配来源、可打开路径 | 只读 |
| 提取上下文 | 当前任务 | `extract` | AI 上下文包、prompt、Markdown | 写新上下文包 |
| 今日提醒 | 可空 | `remind` | 今日 1-3 个重点 | 写新提醒笔记 |
| 检查本地目标 | 本地路径 | `inspect-local-targets` | 路径是否存在、数量、模式 | 只读 |
| 查看文件雷达 | 可空或路径 | `file-radar` | 本地文件报告 | 写新报告 |
| 检查知识库 | 可空 | `obsidian-health` | vault 体检报告 | 写新报告 |
| 二次整理旧资料 | 可空 | `legacy-index` | 旧资料索引 | 写新索引，不移动旧文件 |

## 本地文件 / 目录目标

真实扫描本机文件时，优先粘贴完整路径，例如：

```text
D:\Downloads
D:\Work\report.md
```

拖放和选择文件主要用于告诉 GUI “这些是上下文来源”。浏览器不能直接把本机完整目录权限交给后端，所以需要扫描时仍建议粘贴路径。

## 安全边界

默认不删除、不移动、不重命名、不重写源文件。整理、提取、提醒只写新的 Obsidian 笔记和运行证据；高级诊断只生成报告。
