# 工具维护页重构说明

## 结论

旧首页诊断折叠区不再存在，改为独立的“工具维护页”：`/advanced`。

首页只负责三个主操作：添加资料、搜索回顾、生成 AI 上下文包。工具维护页只放低频但必要的诊断和维护能力，避免用户进入首页后被文件雷达、体检、旧资料索引等次要功能打断。

## 调研依据

- OpenClaw 101 首页采用站点式入口：首屏一句话定位、主 CTA、学习路径、资源卡片流，适合作为本项目首页信息架构参考。
- Progressive disclosure 的核心是先展示必要功能，把低频或高级能力放到第二层，降低复杂应用的学习成本。
- Atlassian 导航案例强调导航应由少量可复用组件支撑，并把动作放在一致位置，减少碎片化入口。
- Atlassian empty state 组件定义强调：没有数据时要说明状态并告诉用户下一步，而不是让用户误以为动作成功。

## 页面分工

### 首页

首页只保留：

- 知识流
- 添加资料
- 搜索回顾
- 生成 AI 上下文包
- 工具维护页跳转入口

首页不再出现：

- 首页诊断折叠区
- 调试 JSON 按钮
- 无解释的旧功能按钮
- 不对应真实后端 action 的入口

### 工具维护页

工具维护页保留 6 个入口：

- 查看文件雷达：只读扫描本地文件或配置目录。
- 检查知识库：只读体检 Obsidian vault。
- 二次整理旧资料：给历史报告和旧笔记建立索引。
- 打开 Obsidian：打开配置的 vault。
- 打开教程 PDF：打开本地教程资料。
- 打开交互说明：打开本地交互说明页。

每个入口必须说明：

- 做什么
- 什么时候用
- 是否写入
- 点击后的结果

## 安全边界

- 默认只读。
- 不删除、不移动、不重命名、不重写源文件。
- 需要写入时，只生成新的 Obsidian 记录或本地运行报告。
- 没有后端 action 的按钮不保留。

## 验收标准

- 首页不出现诊断折叠区。
- `/advanced` 可直接打开。
- 工具页所有按钮都调用真实 `/api/action`。
- 结果区只展示人能读懂的摘要、来源、产物和下一步。
- 无命中或无数据时显示明确空状态和下一步。
